Backing up your SQL Server databases is crucial for data protection, recovery, and business continuity. SQL Server 2025 introduces powerful enhancements to make backup operations faster, more flexible, and less invasive, especially for high-availability environments.

Key Backup Features in SQL Server 2025

1. Offloading Backups to Secondary Replicas

  • Backups on Secondary: A major enhancement in SQL Server Always On Availability Groups lets you perform full, differential, and transaction log backups on secondary replicas, not just COPY_ONLY backups as in previous versions.
  • Benefits: This reduces load on primary servers, enhances performance, and provides operational flexibility. Backup operations can seamlessly continue during failover, improving data protection in disaster recovery setups.
  • Configuration: Use AUTOMATED_BACKUP_PREFERENCE and BACKUP_PRIORITY to specify backup preferences. Enable backup capabilities on secondaries via trace flags and startup parameters.

2. Zstandard (ZSTD) Backup Compression

  • New Compression Algorithm: SQL Server 2025 adds Zstandard, an advanced compression technology that produces smaller backups faster than existing options like MS_XPRESS.
  • Compression Levels: Choose from LOW, MEDIUM, and HIGH levels to balance speed versus compression ratio.
  • Usage: Enable with T-SQL using WITH COMPRESSION (ALGORITHM = ZSTD, LEVEL = ...) during backup commands. Ola Hallengren’s popular backup solution has also been updated to support this.

3. Fast and Efficient Backup Operations

  • Background improvements in backup compression and optimized locking reduce overall backup time and resource consumption.
  • Incremental and differential backups on secondary replicas help minimize backup windows and storage needs.

4. Restore Flexibility

  • Single Table Restore: SQL Server 2025 allows restoring individual tables from backups without requiring full database restores. This capability simplifies recovery from accidental data modifications or deletions.


Best Practices for SQL Server Backups

  • Backup Copies: Always maintain multiple backup copies on different media or locations (on-prem, cloud storage) for safety.
  • Test Restores: Regularly test backup restores to validate recovery procedures.
  • Automate Backups: Use SQL Server Agent jobs or maintenance plans to schedule backups and monitor their health.
  • Secure Backup Storage: Encrypt backup files and secure them to prevent unauthorized access.
  • Optimize Backup Compression: Tune compression level with ZSTD based on your hardware capacity—higher compression reduces storage but requires more CPU.

Example: Performing a Full Backup with ZSTD Compression

BACKUP DATABASE [YourDatabaseName]
TO DISK = 'D:\Backups\YourDB_Full.bak'
WITH COMPRESSION (ALGORITHM = ZSTD, LEVEL = MEDIUM);
GO

Example: Configuring Backups on Secondary Replica

-- Enable differential and full backups on secondary replicas
DBCC TRACEON (3261, -1); -- Differential backups
DBCC TRACEON (3262, -1); -- Full backups
GO

-- Set preference for backups on secondary replicas in Availability Groups
ALTER AVAILABILITY GROUP [YourAGName]
SET (AUTOMATED_BACKUP_PREFERENCE = SECONDARY);
GO

Summary

SQL Server 2025’s backup enhancements improve database resilience, performance, and operational flexibility by:

  • Offloading comprehensive backups to secondary replicas
  • Introducing cutting-edge Zstandard backup compression
  • Allowing single table restores
  • Reducing impact on production workloads

These features empower DBAs to deliver robust data protection while optimizing resource usage and speeding up backup and recovery workflows.
