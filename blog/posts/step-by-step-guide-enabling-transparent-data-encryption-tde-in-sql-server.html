Transparent Data Encryption (TDE) encrypts the underlying database files, protecting data at rest from unauthorized access. Hereâ€™s how you can enable TDE on your SQL Server database:

Prerequisites

  • SQL Server Management Studio (SSMS)
  • Administrative rights on SQL Server
  • A strong password for key protection
  • A secure location to back up your encryption certificate

1. Create a Master Key

The master key encrypts other keys and certificates. Run this in the master database:

USE master;
GO
CREATE MASTER KEY ENCRYPTION BY PASSWORD = 'InsertStrongPasswordHere';
GO

2. Create a Certificate Protected by the Master Key

Certificates are used to protect database encryption keys.

CREATE CERTIFICATE TDE_Cert
WITH SUBJECT = 'Database_Encryption';
GO

3. Create a Database Encryption Key

Switch to your database and create its encryption key, protected by the certificate.

USE [YourDatabaseName];
GO
CREATE DATABASE ENCRYPTION KEY
WITH ALGORITHM = AES_256
ENCRYPTION BY SERVER CERTIFICATE TDE_Cert;
GO

4. Enable Encryption on the Database

Turn on encryption for the database:

ALTER DATABASE [YourDatabaseName]
SET ENCRYPTION ON;
GO

5. (Recommended) Back Up the Certificate & Private Key

This is crucial for future restores or migrations. Run in master:

BACKUP CERTIFICATE TDE_Cert 
TO FILE = 'C:\TDE_Cert.cer'
WITH PRIVATE KEY (
    FILE = 'C:\TDE_Cert_Key.pvk',
    ENCRYPTION BY PASSWORD = 'AnotherStrongPassword'
);
GO

Store these files and passwords securely!

6. Verify Encryption Status

Run the following to check the status of database encryption:

SELECT db_name(database_id) AS DatabaseName,
       encryption_state,
       key_algorithm,
       key_length
FROM sys.dm_database_encryption_keys;
GO

Encryption State:

  • 0 = No encryption
  • 1 = Unencrypted
  • 2 = Encryption in progress
  • 3 = Encrypted

Tips & Best Practices

  • Always back up your certificate and private key immediately after creation.
  • Keep multiple secure copies for disaster recovery and migration scenarios.
  • You must have the certificate and private key to restore or attach your encrypted database on another server.
  • TDE does not encrypt data in transitâ€”use SSL for client connections if needed.

With these steps, your SQL Server database will be protected with TDE, meeting compliance and data security requirements for encryption at rest.
